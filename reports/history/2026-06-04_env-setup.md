# Phase 1 시작: GPU 환경 검증 + retriever stack 결정

## What changed

- 사용자가 본 Pod 의 GPU 4 × NVIDIA H100 80GB HBM3 (compute capability
  9.0) 를 사용 가능하다고 확인. 4대 모두 인식 (`nvidia-smi`).
- 다음 환경 핀이 검증되어 `reports/05_reproducibility.md §2` 와
  `requirements.txt` 에 반영됨:
  - Python 3.12.3 (system-wide `/usr/local/bin/python`)
  - CUDA toolkit 12.8 (`/usr/local/cuda-12.8`)
  - GPU driver 535.247.01 (driver bundled CUDA 12.2 runtime)
  - PyTorch 2.9.1 (CUDA 12.8 wheel)
  - cuVS 26.04 + pylibraft 26.04 + cupy-cuda12x 14.1.1
  - faiss-cpu 1.14.2 (CPU baseline 전용)
  - sentence-transformers 5.5.1, transformers 5.8.1, datasets 4.8.4
  - scikit-learn 1.9.0, nvidia-ml-py
- **Retriever stack 의 큰 변경**: 원래 `FAISS-GPU IVF-PQ` 를 GPU baseline
  의 메모리 효율 측으로 쓸 계획이었으나, `faiss-gpu-cu12` pip wheel 이
  PyTorch 2.9 의 cublas 12.8 ABI 와 충돌 (`undefined symbol:
  cublasLtGetEnvironmentMode`). 해당 wheel 을 제거하고 cuVS 가 동일
  알고리즘 (IVF-PQ) 을 제공하므로 **`cuvs.neighbors.ivf_pq` 로 대체**.
- 위 변경에 따라 다음 파일들이 갱신됨:
  - `configs/retrievers/faiss_ivfpq_gpu.yaml` → `cuvs_ivfpq.yaml` (param
    이름도 cuVS API 에 맞게 `nlist`→`n_lists`, `nprobe`→`n_probes`,
    `m`→`pq_dim`, `nbits`→`pq_bits`).
  - `configs/experiments/phase1_baseline.yaml` 의 retriever 리스트.
  - `reports/03_baseline_methodology.md §3.1` 의 retriever matrix.
- `reco_bench/utils/{seed,io,gpu_info}.py` 추가 — 학습/평가 파이프라인
  의 공통 base.

## Why

- **cuVS 로 GPU 측 통일** 의 장점:
  1. PyTorch + cuVS 의 dependency conflict 완전 해소.
  2. CAGRA 와 IVF-PQ 둘 다 같은 라이브러리 (cuVS 26.04) 안에서 — 동일
     hardware 캡처, 동일 측정 프로토콜 보장.
  3. cuVS 는 NVIDIA RAPIDS 의 long-term 지원 라이브러리이며, CUDA 12.8
     까지 stable wheel 제공. faiss-gpu 는 CUDA 12.x 에서 wheel 지원이
     불안정.
- **CPU-side numpy 반환 contract 와 무관**: cuVS 도 `cupy.asnumpy()` /
  `__cuda_array_interface__` 를 통해 CPU 반환 가능. `Retriever.search`
  의 약속을 깨지 않는다.

## Validation

- `nvidia-smi` 4× H100 확인 (driver 535, 모두 80GB HBM3).
- PyTorch CUDA matmul on `cuda:0` 성공: `(100, 100) @ (100, 100)`.
- cuVS CAGRA build + search smoke test 통과: 1024 items × 128 dim,
  build 1052 ms, search 98 ms for 10 queries.
- cuVS IVF-PQ 도 `from cuvs.neighbors import ivf_pq` import 성공.
- faiss-cpu HNSW smoke 통과: 1000 items × 128 dim, top-5 정상 반환.
- `reco_bench.utils.seed.set_global_seed` 가 numpy/torch/PYTHONHASHSEED
  모두 일관 고정. `gpu_info.capture_hardware` 로 43 KB hardware dump
  생성 가능.
